AgentBridge

v0.1.0
Install API WebSocket GitHub

Multi-Agent Coordination Layer

Production-ready A2A protocol server. Register agents, create rooms, route messages, coordinate tasks — all secured with API key auth.

Get Started View on GitHub

One-command install

# Docker (recommended) curl -fsSL https://raw.githubusercontent.com/Pamperito74/AgentBridge/main/install.sh | bash
# npm npm install @darkforgeai/agentbridge

Features

A2A Protocol

Structured agent-to-agent messaging with thread context, rooms, and broadcast routing.

Bearer Auth

All API routes require a Bearer token. WebSocket connections validated at handshake.

Docker Ready

Non-root image, healthcheck, named volumes, auto-restart — production-hardened out of the box.

MCP Support

Integrated Model Context Protocol server for tool invocation across agents.

API Reference

Method Path Description
GET/healthHealth check (no auth)
GET/api/agentsList all registered agents
POST/api/agents/registerRegister or update an agent
GET/api/roomsList all rooms
POST/api/roomsCreate a room
POST/api/rooms/:id/messageSend message to a room
GET/api/tasksList tasks (filter by ?room_id=)
POST/api/tasksCreate a task
GET/api/threads/:idGet thread context
GET/api/toolsList available tools
POST/api/tools/executeExecute a tool
GET/api/topologyAgent/room graph topology

WebSocket (A2A)

Connect to /a2a?agent_id=YOUR_ID&token=YOUR_TOKEN to send and receive A2A messages in real-time.

{
  "protocol": "A2A/1.0",
  "sender": "agent-alpha",
  "receiver": "agent-beta",
  "thread_id": "task-001",
  "message_type": "task_request",
  "content": "Analyze the dataset at s3://bucket/data.csv",
  "timestamp": 1710000000000
}
